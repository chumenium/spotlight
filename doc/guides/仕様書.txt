SpotLight アプリ フロントエンド仕様書（AIプログラマー向け要約）
1. アプリ概要

名称: SpotLight（隠れた才能発見プラットフォーム）

コンセプト:
日常の中の小さな成果や才能を気軽に投稿・共有し、ユーザー同士で承認（いいね/コメント/バッジ）するSNS型アプリ。

2. 対象ユーザー

才能や日常の工夫を共有したい個人ユーザー

共感や承認を得たいユーザー

ゲーミフィケーション要素を楽しみたいユーザー

3. 基本機能（フロントエンド側）
3.1 ユーザーアカウント関連

ログイン / ログアウト / 新規登録

プロフィール編集（固定ニックネーム、アイコン画像、自己紹介）

マイページ表示（自分の投稿・バッジ・フォロワー数）

3.2 投稿機能

テキスト投稿（短文）

画像投稿（複数枚対応）

タグ付け（カテゴリ分類）

投稿編集 / 削除

投稿詳細画面（コメント欄、いいね数、バッジ情報）

3.3 承認・コミュニケーション機能

いいね（単一クリック）

コメント（ツリー構造不要、平文リスト表示）

バッジ付与（条件達成時に自動表示）

3.4 バッジ / ランク機能

条件: 投稿数、いいね数、フォロワー数など

フロント側は「バッジ画像の表示」と「獲得履歴の一覧」処理のみ

3.5 タイムライン

投稿フィードの一覧表示（新着順 / 人気順）

フィルター（タグ / ユーザー単位）

無限スクロール or ページネーション

3.6 通知機能

いいね / コメント / バッジ獲得時に通知一覧に表示
（リアルタイムは不要。ポーリング or 画面更新時取得）

4. 画面一覧（フロントエンド想定）

ログイン/新規登録画面

ホーム/タイムライン

投稿作成画面

投稿詳細画面

プロフィール画面

マイページ

通知一覧画面

設定画面

5. UI仕様（主要ポイント）

色味: シンプルで温かみのある配色（ライトテーマ）

投稿カード形式: 画像 + テキスト + タグ + いいね/コメントアイコン

バッジ表示: プロフィールアイコン横に小さく重ね表示

レスポンシブ対応: スマホ優先（Web/アプリ共通）

6. 技術要件（フロントエンド）

使用フレームワーク: React or Vue（どちらか選定）

状態管理: Redux / Vuex / Zustand など

UIライブラリ: Material UI / TailwindCSS

画像アップロード機能: フロント側でプレビュー表示必須

API通信: REST API (JSON形式) を前提とした axios/fetch 実装

認証: JWT想定（ログイン後にトークンを保持）

7. 非機能要件（フロント側）

モバイルファーストデザイン

読み込み速度を意識した軽量化

エラーハンドリング（API失敗時のアラート/リトライ）

多言語対応は初期リリースでは不要
